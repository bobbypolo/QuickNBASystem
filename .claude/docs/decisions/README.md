# Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) documenting significant technical decisions made in this project.

## What is an ADR?

An ADR captures the context, options considered, and rationale for important technical decisions. This helps future contributors understand _why_ things are the way they are.

## When to Create an ADR

Create an ADR when:

- Choosing between competing technologies or libraries
- Making architectural changes that affect multiple components
- Establishing patterns or conventions for the codebase
- Making decisions that are hard to reverse

## How to Create an ADR

Use the `/decision` skill or manually copy `000-template.md`.
